## 赛制特点

蓝桥杯和 ACM 赛制有很大不同，了解赛制是备赛的第一步：

- **填空题**（5 道）：只需要提交答案，可以暴力、打表、手算
- **编程题**（5 道）：需要写完整代码，按测试点给分
- **时长**：4 小时
- **语言**：C/C++、Java、Python

蓝桥杯最大的特点是**按测试点给分**，这意味着即使做不完也能拿部分分。所以策略很重要。

## 备赛时间表

### 赛前 2 个月

系统复习核心知识点：

- 搜索（BFS/DFS/回溯）
- 动态规划（背包、线性 DP）
- 图论（最短路、并查集）
- 数学（排列组合、GCD、素数）
- 字符串处理

### 赛前 1 个月

开始刷历年真题。蓝桥杯真题是最好的练习材料，因为风格和难度都比较固定。

建议按年份倒序刷，最近 3 年的真题最有参考价值。

### 赛前 1 周

- 整理常用代码模板
- 复习易错点
- 模拟考试（严格计时 4 小时）

## 做题策略

### 填空题

填空题是送分题，一定要全拿：

```cpp
// 暴力枚举示例：求满足条件的数
#include <iostream>
using namespace std;

bool check(int n) {
    // 检查条件
    while (n > 0) {
        int d = n % 10;
        if (d == 2 || d == 0 || d == 2 || d == 6) return false;
        n /= 10;
    }
    return true;
}

int main() {
    int cnt = 0;
    for (int i = 1; i <= 2026; i++) {
        if (check(i)) cnt++;
    }
    cout << cnt << endl;
    return 0;
}
```

填空题不需要考虑效率，能出答案就行。手算、Excel、Python 交互式都可以上。

### 编程题

编程题的策略：

1. **先拿部分分**：先写一个暴力解，确保拿到 30-50% 的测试点分
2. **再优化**：有时间再写正解
3. **边界检查**：特别注意 n=0、n=1、空图等特殊情况

```cpp
// 暴力解：先拿部分分
void solve_brute() {
    // O(n^2) 或 O(2^n) 的暴力
}

// 正解：有时间再写
void solve_optimal() {
    // O(n log n) 的正解
}
```

## 常用模板

考前一定要整理好模板，比赛时直接用：

```cpp
// 快速幂
long long qpow(long long a, long long b, long long mod) {
    long long res = 1;
    a %= mod;
    while (b > 0) {
        if (b & 1) res = res * a % mod;
        a = a * a % mod;
        b >>= 1;
    }
    return res;
}

// 并查集
int parent[100005];
int find(int x) { return parent[x] == x ? x : parent[x] = find(parent[x]); }
void unite(int x, int y) { parent[find(x)] = find(y); }

// BFS 最短路
int dx[] = {0, 0, 1, -1};
int dy[] = {1, -1, 0, 0};
```

## 考场注意事项

1. **读题要仔细**：蓝桥杯的题目描述有时比较长，别漏条件
2. **开 long long**：数据范围不确定时，默认开 `long long`
3. **多组数据**：注意题目是否有多组测试数据
4. **文件输入输出**：有些题要求文件读写，别忘了
5. **编译检查**：交卷前编译运行一遍，检查输出格式

## 总结

蓝桥杯相比 ACM，对算法深度的要求没那么高，但对**代码实现能力**和**做题策略**要求更高。把真题刷透、模板备好、策略得当，拿省一并不难。
